# Information Security Policy (draft v0.1)

**Status:** Draft — not yet approved for audit  
**Owner:** TBD  
**Review cycle:** Annual  

## Purpose

Define security requirements for MapAble systems processing participant and provider data.

## Scope

All MapAble production systems, personnel with access, and subprocessors listed in [subprocessor-register.md](../subprocessor-register.md).

## Policy statements (to be expanded)

1. Access is granted on least-privilege basis with RBAC.  
2. Authentication is required for all non-public APIs.  
3. Audit events are recorded for sensitive actions.  
4. NDIS participant numbers are encrypted at rest.  
5. Security incidents are reported per the Incident Response Plan.  
6. Vendors are assessed before processing organisational data.
