# Care module (Phase 3)

## Built
Care requests, care shifts, worker assignment, check-in/out, participant approval, admin/provider workflows.

## Routes
`/dashboard/care`, `/api/care/requests`, `/api/care/shifts`, `/admin/care`, `/provider/care`

## Privacy
Accessibility details require consent (`care.accessibility_share`). Participants see access notice on forms.

## Limitations
No AI matching, GPS check-in, or recurring bookings.

## Phase 4
Rostering, timesheets, incident reporting.
